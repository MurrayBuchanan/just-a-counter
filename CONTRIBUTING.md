# Contributing to Just a Counter

Thanks for taking the time to contribute!

## How to Contribute

### Reporting Bugs

Open an issue and include:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Browser/OS if relevant

### Requesting Features

Open an issue with the `enhancement` label. Describe the use case — not just the feature itself.

### Submitting Code

1. Fork the repo
2. Create a branch: `git checkout -b feat/your-feature` or `fix/your-bug`
3. Make your changes
4. Open a pull request against `main`

Use the PR template provided — fill it in properly or it'll be closed.

### Improving Documentation

Typos, unclear wording, missing context — all welcome. Same process as code: fork, branch, PR.

## Branch Naming

| Type | Pattern |
|------|---------|
| Feature | `feat/short-description` |
| Bug fix | `fix/short-description` |
| Docs | `docs/short-description` |

## Pull Request Template

When you open a PR, you'll be prompted with:

```
## What does this PR do?

## Why is this change needed?

## How has it been tested?

## Checklist
- [ ] Code works locally
- [ ] No unnecessary files included
- [ ] PR description is filled in
```

## Code Style

Keep it simple — this is a counter app. If you're unsure whether something fits the project's scope, open an issue first.
